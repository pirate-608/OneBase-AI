# Database Configuration

The `database` field defines the database type and vector storage engine used by OneBase.

---

## Basic Structure

```yaml title="onebase.yml"
database:
  type: postgresql
  vector_store: pgvector
```

| Field          | Type   | Only Valid Value | Description           |
| :------------- | :----- | :--------------- | :-------------------- |
| `type`         | string | `postgresql`     | Relational database   |
| `vector_store` | string | `pgvector`       | Vector storage engine |

!!! warning "No extensions supported yet"
    OneBase currently only supports the **PostgreSQL + pgvector** combination. Specifying other values (e.g., `mysql`, `milvus`) will cause a configuration error on startup. Future versions may add support for more databases and vector engines.

---

## Connection Credentials

Database connection details are configured via `.env` environment variables, **not in `onebase.yml`**:

```bash title=".env"
POSTGRES_USER=onebase              # Username
POSTGRES_PASSWORD=xxxxxxxx         # Password (auto-generated by onebase init)
POSTGRES_DB=onebase_db             # Database name
POSTGRES_HOST=localhost            # Optional, defaults to localhost
POSTGRES_PORT=5432                 # Optional, defaults to 5432
```

`onebase init` automatically generates a secure random password and writes it to `.env`. Usually no manual changes are needed.
